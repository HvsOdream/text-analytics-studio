역할 지정 (Role Prompting)

AI에게 "누구로서" 대답할지를 알려준다

왜 역할을 지정하면 결과가 달라지는가?

AI는 방대한 텍스트 데이터에서 학습했기 때문에, "누구의 관점"으로 답해야 하는지를 알려주면 그 분야의 어휘·깊이·관점이 활성화됩니다.

  • 역할 없이 질문 → 일반적이고 피상적인 답변
  • 역할 지정 후 질문 → 해당 전문가의 어조, 프레임, 전문 용어가 등장

비교: 역할 없음 vs 역할 있음

❌ 역할 없음 
학생들의 글쓰기 능력을 분석하고 싶어. 방법을 알려줘.
✅ 역할 지정 
너는 20년 경력의 초등학교 국어 교사이자, 서술형 평가 전문가야.
내가 6학년 학생 30명의 독서감상문을 줄 거야.
이 글에서 학생들의 글쓰기 역량을 분석하는 방법을 알려줘.
💡 역할 지정의 3요소: ① 직함/경력 ("20년 경력 국어 교사") + ② 전문 분야 ("서술형 평가 전문가") + ③ 맥락 ("6학년 학생 30명")
상황추천 역할기대 효과
학생 답안 분석교과 전문 교사 + 평가 전문가교과 특성을 반영한 구체적 채점 기준
수업 피드백 분석교육과정 전문가 + 데이터 분석가패턴 발견 + 구조화된 인사이트

같은 질문도 "자유롭게 답해줘"와 "표 형식으로 정리해줘"는 완전히 다른 결과를 만듭니다. 형식을 지정하면:

단계적 분석 (Chain-of-Thought)

복잡한 작업을 쪼개서 정확도를 높인다

왜 한 번에 시키면 틀릴까?

"이 30개 답안을 읽고, 분류하고, 채점하고, 피드백까지 작성해줘"라고 하면 — AI는 중간 과정을 건너뛰거나, 앞뒤가 안 맞는 결과를 줍니다.

Chain-of-Thought(CoT)는 AI에게 "생각하는 과정"을 단계별로 밟게 해서, 복잡한 작업의 정확도를 높이는 기법입니다.

비교: 한 번에 vs 단계별

❌ 한 번에 모든 것을 요청 
아래 학생 에세이를 읽고 채점하고 피드백을 작성해줘.
채점 기준은 논리성, 근거, 표현력이야.
✅ 단계별 분석 지시 
아래 학생 에세이를 분석해줘. 다음 단계를 순서대로 수행해:

[1단계] 먼저 에세이의 핵심 주장을 한 문장으로 요약해줘.
[2단계] 핵심 주장을 뒷받침하는 근거를 모두 추출해줘.
[3단계] 다음 3개 항목으로 채점해줘 (각 10점 만점):
  - 논리적 구성
  - 근거의 적절성
  - 표현력
[4단계] 각 항목별로 구체적인 피드백을 작성해줘.
  - 잘한 점 1가지, 개선할 점 1가지씩.
[5단계] 학생에게 전달할 종합 코멘트를 3문장으로 작성해줘.
⚠️ 핵심 원칙: 단계 번호를 [1단계], [2단계]로 명시하세요. AI가 순서를 건너뛰는 것을 방지합니다. 각 단계의 결과를 다음 단계의 입력으로 쓸 때 가장 효과적입니다.
패턴 A: 읽기 → 추출 → 분류 → 해석

텍스트를 먼저 읽고, 핵심 요소를 추출한 뒤, 카테고리별로 분류하고, 의미를 해석

적합: 성찰일지, 수업 피드백, 상담 기록

패턴 B: 기준 설정 → 적용 → 채점 → 피드백

먼저 채점 기준을 확인하고, 답안에 적용한 뒤, 점수를 매기고, 피드백 작성

적합: 서술형 채점, 포트폴리오 평가

패턴 C: 비교 대상 정리 → 차이점 → 공통점 → 시사점

두 개 이상의 텍스트를 비교할 때, 각각 정리 → 차이점 → 공통점 → 종합

적합: 교육과정 비교, 교재 비교, 전후 비교

패턴 D: 전체 요약 → 세부 분석 → 패턴 발견 → 제안

큰 그림부터 보고, 세부를 파고, 패턴을 찾고, 행동 제안으로 마무리

적합: 대량 설문 분석, 학기 전체 데이터 분석

Few-shot 예시 제공

원하는 답의 샘플을 보여주고 "이런 식으로 해줘"

Few-shot이 강력한 이유

"이렇게 해줘"라고 설명하는 것보다, "이게 예시야, 이런 식으로 해줘"가 훨씬 정확합니다.

  • Zero-shot: 예시 없이 지시만 → 일반적인 결과
  • One-shot: 예시 1개 → 패턴을 어느 정도 파악
  • Few-shot (2~5개): 예시 여러 개 → 패턴을 정확히 학습

비교: Zero-shot vs Few-shot

❌ Zero-shot (예시 없음) 
아래 학생 답안의 깊이를 1~5점으로 채점해줘.

[학생 답안]
"인공지능은 미래에 중요할 것 같습니다."
✅ Few-shot (예시 3개 제공) 
아래 학생 답안의 깊이를 1~5점으로 채점해줘.
채점 기준은 다음 예시를 참고해:

[예시 1]
답안: "인공지능은 좋은 것 같습니다."
점수: 1점
이유: 구체적 근거 없이 막연한 느낌만 표현

[예시 2]
답안: "인공지능은 교육에서 개인화 학습을 가능하게 한다. 하지만 데이터 편향 문제를 해결해야 한다."
점수: 4점
이유: 구체적 활용 사례 + 한계점을 함께 제시

[예시 3]
답안: "인공지능의 자연어처리 기술은 학생의 서술형 답안을 분석하여 교사의 채점 부담을 줄일 수 있다. 다만 2023년 UNESCO 보고서에 따르면..."
점수: 5점
이유: 구체적 기술명 + 실증적 근거 + 출처 제시

이제 아래 답안을 같은 방식으로 채점해줘:
[학생 답안]
"인공지능은 미래에 중요할 것 같습니다."
💡 Few-shot 황금 규칙: ① 예시는 2~5개가 최적 (너무 많으면 혼란) ② 좋은 예시와 나쁜 예시를 섞어라 ③ 채점 이유를 반드시 포함하라
Step 1: 앵커 포인트 설정

채점 척도의 양 끝(최고점/최저점)에 해당하는 답안을 먼저 예시로 만든다.

Step 2: 중간 지점 추가

중간 점수(3점 등)에 해당하는 예시를 1~2개 추가한다. AI가 "경계선"을 학습하는 데 도움.

Step 3: 이유 명시

각 예시에 왜 그 점수인지 이유를 써준다. 이유가 없으면 AI는 점수 패턴만 따라하고 논리를 모방하지 못한다.

결과: [기대 출력 1]
이유: [왜 이런 결과인지]
